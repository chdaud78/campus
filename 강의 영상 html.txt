이미지 종류
레스터(Raster / 비트맵이라고 함) vs 벡터(Vector)

비트맵 : 픽셀이 모여 만들어진 정보의 집합(.jpeg, .gif, .png 등)
 ㄴ 정교하고 다양한 색상을 자연스럽게 표현
 ㄴ 확대/ 축소시 계단현상, 품질저하
 ㄴ 일반적인 풍경 사진들 등


벡터 : 점선면의 위치(좌표) 색상등/ 수학적 정보의 형태로 이루어진 이미지(.svg)
 ㄴ 확대 축소에서 자유로움, 용량 변화 x
 ㄴ 정교한 이미지(인물 풍경)을 표현하기 어려움
 ㄴ 아이콘/ 로고 등의 Material Design(고품질 디지털 경험을 구축할 수 있도록 google에서 만든 디자인 시스템) 에 용이

필요에 따라 어떤 이미지를 선택하는가가 중요


JPG
 ㄴ FULL-COLOR와 GRAY-SCALE의 압축을 위해 만들어짐, 압축률이 훌륭해 사진이나 예술에서 많이 사용
 ㄴ 손실 압축(이미지가 손실 되면서 압축이 됨) -> 용량이 줄어듬
 ㄴ 이미지 반복적 저장시 계속 손실됨
 ㄴ 표현 색상도 24비트(1600만 색상) 뛰어남
 ㄴ 이미지 품질과 용량 쉽게 조절
 ㄴ 가장 널리 쓰임

PNG 
 ㄴ GIF의 대체 포멧으로 개발
 ㄴ 비손실 압축 -> 용량이 큼
 ㄴ 8비트 (256색상) / 24비트(1600만 색상) 컬러 이미지 처리
 ㄴ ALPHA CHANNEL 지원(투명도 -> 이미지 주변 흰색 안나옴)
 ㄴ W3C 권장 포맷

GIF
 ㄴ 이미지 파일 내에 이미지 문자열 같은 정보들을 저장
 ㄴ 비손실 압축
 ㄴ 여러 자의 이미지를 한개의 파일에 담읈 있음
 ㄴ 애니메이션
 ㄴ 8비트 색상만 지원(다양한 색상 지원 X)
 
WEBP
 ㄴ JPG, PNG, GIF를 모두 대체할 수 이는 구글이 개발한 이미지 포맷
 ㄴ 완벽한 손실/ 비손실 압축 지원
 ㄴ GIF 같은 애니메이션 지원
 ㄴ ALPHA CHANNEL 지원(손실 비손실 모두)
 ㄴ 단 IE 지원 불가

하위 호환성
 ㄴ 하위 브라우저에서 호환이 가능한가를 확인

SVG 
 ㄴ 마크업 기반의 벡터 그래픽을 표현하는 포맷
 ㄴ 해상도에서 자유로움
 ㄴ CSS, JS로 제어가능
 ㄴ 파일 및 코드 삽입 가능

HTML 태그 -> 문서의 전체 범위
HEAD 태그 -> 문서 정보를 나타내는 범위 (제목, 설명, 파일 위치, css 등의 정보)
BODY 태그 -> 문서의 구조를 나타내는 범위(사용자 화면에 보여지는 웹페이지의 보여지는 구조를 작성 )
TITLE 태그 -> 문서의 제목을 정의
LINK 태그 -> 외부 문서(대부분 css)를 가져와 연결 (필수 속성 : rel - 문서와의 관계, href - 경로)
STYLE 태그 -> css를 HTML 문서 내에서 작성하는 경우
SCRIPT 태그 -> JS를 가져오는 태그(1. 외부 파일 가져오기 2. JS 문법을 HTML 문서 내에서 작성)
META 태그 -> HTML 문서의 제작자, 내용 키워드 등의 정보를 검색엔진, 브라우저에 제공 (name : 정보의 종류, content : 정보의 값)
IMG 태그 -> 이미지 태그(Alt 속성 : 대체 텍스트)

글자와 상자
요소가 화면에 출력되는 특성, 크게 2가지로 구분
인라인(inline) 요소
 ㄴ 글자를 만들기 위한 요소들
 ㄴ 수평
 ㄴ 가로, 세로 콘텐츠 크기만큼 자동으로 줄어듬
 ㄴ 가로/세로 사이즈 지정x
 ㄴ 위 아래 여백 불가/ 좌우는 가능
 ㄴ 글자 안에 상자는 불가 (인라인 요소 안에 블록 요소 불가)
 ㄴ 대표적 인라인 요소 -> span : 아무것도 나타내지 않고 콘텐츠 영역을 설정하는 용도
 ㄴ cf) <span></span><span></span> 과 <span></span> 은 다름 앞은 띄어쓰기 x, 뒤는 띄어쓰기 들어감. 즉 줄바꿈은 띄어쓰기로 인식
                                     <span></span>
블록(block) 요소
 ㄴ 상자(레이아웃)를 만들기 위한 요소들
 ㄴ 수직
 ㄴ 가로는 부모요소 크기만큼 자동으로 늘어남, 세로는 콘텐츠 크기만큼 자동으로 줄어듬.
 ㄴ 가로/세로 사이즈 지정 o
 ㄴ 위/아래/좌/우 모두 여백 가능
 ㄴ 블록 요소 안에 블록 가능 물론 인라인도 가능
 ㄴ 대표적 블록 요소 -> div : 아무것도 나타내지 않고 콘텐츠 영역을 설정하는 용도

 인라인 블록
 ㄴ 베이스는 인라인
 ㄴ 블록 요소의 특성을 가짐.

자주 사용 하는 요소
블록
div - division 특별한 의미가 없는 구분을 위한 요소
h1 - heading 제목을 의미(1~6 숫자가 작을수록 더 중요한 제목)
p - paragraph 문장을 의미
ul - unordered list 순서가 필요없는 목록의 집합
li - list item 목록 내 각 항목


인라인
span - 특별한 의미가 없는 구분을 위한 요소
img - image 이미지 (필수 속성 : 삽입-src 경로 / alt-이름)
a - anchor 다른/같은 페이지로 이동하는 하이퍼링크를 지정하는 요소(href-url / target-링크 url 표시 위치)
br - break 줄바꿈
label - 라벨 가능 요소(input)의 제목

inline-block
input - 사용자가 데이터를 입력하는 요소
 ㄴ (type-입력받을 데이터의 타입 / value-미리 입력된 값(데이터)/ placeholder-사용자가 입력할 값의 힌트 / disabled-입력 요소 비활성화 / name-이름의 그룹)

table 요소
table - 표, 행(row)과 열(column)의 집합
tr - table row 행
td - table data 열

전역 속성
title - 요소의 정보나 설명을 지정, 해당 요소에 마우스를 올리면 툴팁처럼 보임.
style - 요소에 적용할 스타일을 지정(인라인 스타일이라 부름)
class - 요소를 지칭하는 중복 가능한 이름 (선택자, css/js로 제어하기 위해 주로 이용)
id - 요소를 지칭하는 고유한 이름
data - 요소에 데이터를 지정
 ㄴ ex) data-이름="데이터"
